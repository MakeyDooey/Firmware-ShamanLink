(UM11931 User Manual from NXP docs)
MCU-Link Base standalone Debug Probe

Main Features
 - UART to USB bridge (VCOM) 
 - CMSIS-DAP firmware to support all NXP Arm Comtex-M based MCUs with SWD debug interfaces 
 - High speed USB host interface 
 - SWO profiling and I/O features
 - CSMIS-SWO support

Board Layout

J1: Host USB connector
J2: LPC55S69 SWD connector (for development of custom debug probe code only)
J3: Firmware update jumper (reinstall and re-power to update firmware)
J6: SWD connector for connection to tagret system 
J7: VCOM connection 
J8: Digital expansion connector

Installation and Firmware Options
 - Factory programmed with NXP's CMSIS-DAP protocol based firmware
 - This model of MCU-Link cannot run the version of J-Link firmware from SEGGER that is available for other MCU-Link
 - Some early product units may not have a debug probe firmaware image installed. If this is the case none of the LEDs will light up when the board is connected to a host computer. // currently the case

Manually updating MCU-Link firmware 
 1. It must first be powered up in the ISP mode. To do this, insert jumper J3 and connect MCU-Link to your host computer using a micro-B USB
 2. The red STATUS LED (LED3) should light up and stay on. 
 3. The board enumerates on the host computer an HID class USB device * 
	- Note: from version V3.xxx onwards, the MCU-Link firmware uses WinUSB instead of HID for higher performance 	(requires MCUXpresso >=11.7) (MCUXpresso for VS Code versions recommended)
 4. Nagivate to the MCU-LINK_installer\scripts subdirectory
	- install LinkServer to get these scripts // download link doesn't exists anymore
	- Recent MCUXpresso IDE versions integrate LinkServer as a stand-alone product installed in a folder located 	at the same level as the IDE product (C:\nxp\LinkServer_25.6.131)
 5. Run the program_CMSIS.cmd script
 6. Follow nagivation instructions from the script
 7. Unplug the board, remove J3, and reconnect the board 
 8. After programming the MCU-Link with the CMSIS-DAP firmware, a USB serial bus device and a virtual com port will enumerate. The status LED repeatedly fades from on to off and back again ("breathing")

PRD: Professional USB-CLI control system

1. Hardware Specification & Pin Mapping

The system targets the LPC55S69JBD64 (U1). Control of the RGB LED is achieved via three dedicated GPIO pins on Port 1. Because the hardware is wired in an Active-Low configuration (Common Anode), the GPIO must sink current (Logic 0) to illuminate the LED.

Signal	GPIO	Schematic Pin	Active Level	SDK Constant
Red	PIO1_1	39		Low (0=ON)	BOARD_LED_RED_GPIO_PIN
Blue	PIO1_2	41		Low (0=ON)	BOARD_LED_BLUE_GPIO_PIN
Green	PIO1_3	42		Low (0=ON)	BOARD_LED_GREEN_GPIO_PIN

1.1 Clocking & USB PHY Configuration

- Reference Clock: External 32MHz Crystal (Y1) on Pins 19/20.
- USB Controller: USB1 (High-Speed IP).
- PHY Requirements: The USB1 PLL must be engaged to provide the 480MHz PHY clock.
- Clock Tree: Main Clock at 150MHz (PLL0); USB1 High Speed Clock sourced from the USB PLL.
- VBUS Sensing: Mandatory sensing on Pin 26 (USB1_VBUS). USB_DeviceCdcVcomInit must use physical VBUS detection to support hot-plugging.

2.2 Task Priority & Scheduling

Task Name	Priority	Stack Size	Responsibility
USB_Device_Task	4 (High)	2048		USB stack maintenance and endpoint ISR handling.
CLI_Task	3		4096		String parsing, command execution, and LED GPIO control.
Idle_Task	0		-		System sleep / low-power entry.

2.3 Buffer Strategy

- Mechanism: StreamBufferHandle_t (Thread-safe FIFO).
- RX Buffer (512B): USB ISR pushes raw bytes here via xStreamBufferSendFromISR.
- TX Buffer (1024B): CLI task pushes response strings; USB task pulls and sends via USB_DeviceCdcAcmSend.

How RS-845 works: 	

(Wikipedia) RS-485 is an electrical signaling standard using differential signaling over twisted pair. It is generally accepted that it can be used with data rates up to 10Mbit/s, or the product of data rate and distance should not exceed 10^8. The RS-485 uses three-state logic allowing individual transmitters to be deactivated. This allows RS-485 to implement linear bus topologies using only two wires.

    Linear bus topology				  Star topology			     Ring topology

           (node)        (node)                       (node)	                     ___(node)___		
             |             |                            |                           /            \
(root) ------+------+------+                 (node)---(root)---(node)	         (node)        (node)	
                    |                                 /    \                        \            /
                  (node)                          (node)  (node)                  (node)------(node)

Mode of Operation

Different receiver levels:
 - binary 1 (OFF): Voa - Vob < -200mV ~= Voa < Vob
 - binary 0 (ON):  Voa - Vob > +200mV ~= Voa > Vob

Signal	Mark (logic 1) 	Space (logic 0)
A	Low		High
B	High 		Low

Never set these in software, powering any of these will result in the ShamanLink being un-programmable. If this ever occurs, see the "Hard Reset" instructions to mass flash erase the board.

 - ProSWDCLK: wired to PIO0_11 / ADC0_9
